Manual/Inline Events

These links have inline GAEvent code on it. Click one of these links, and you should see it show up in GA.

<a 
  href="/link-destination" 
  data-event="GAEvent" 
  data-category="Example_Category" 
  data-label="Example_Label" 
  data-action="Click" 
  data-value="undefined"
>
  Link Text
</a>



Auto-Tagger Events

These links are set up using the auto-tagger code. Instead of setting the events using inline code, it is set with a single snippet of JS that adds the code to each element and sets the label based on the text or URL. This can be useful for easily tagging all elements in a section, or tagging elements used across multiple sections/pages.

  • test1|
  • test2|
  • test3|
  • test4

Example:

var category  = "Nav";
var action  = "Click";
var value  = "";
var label_format   = "";

$('#nav a').autotagger(
  category, 
  action, 
  value, 
  label_format
);



UTM Parameter Events

The analytics suite includes support for persistent UTM parameters via JS sessionStorage.

[Experimental] - duplicate sessionStorage code with localStorage for long-term persistence.

[Experimental] - log UTM parameters as GA events.

Simple UTM link:
/?utm_source=test&utm_medium=test&utm_campaign=test&utm_term=test&utm_content=test

Google's Campaign URL Builder - use this to easily create URLs with UTM parameters. Also, I don't really feel like building that tool from scratch.




UTM tester

This section will display your current UTM values. Use this tool to test/demo the persistent UTM functions across pages.

[Experimental] - utm_state: this variable shows the current state of the UTM parameters, and can have 3 possible values. Note that this UTM field is non-standard.

  • "url" - The UTM params are present in the URL
  • "sessionStorage" - The UTM params are present in the user's session
  • "localStorage" - The UTM params are present in the user's long-term cookie storage

Form Events

This form is set up with GA events triggered by the form submission, not on just clicking the submit button. You should only see the success event fire when the form successfully submits, otherwise you'll get false positive events.

[Experimental] - Set up event for partially filled/abandoned forms.

[Experimental] - Automatically adds and auto-fills hidden form fields for the user's UTM values.

Scroll/InView Events

This section is set up to trigger an event when it is visible on screen. In the past, this would require use of heavy JS code, but Google Analytics 4 supports this type of event natively, now.

This element is tagged for a scroll event.
